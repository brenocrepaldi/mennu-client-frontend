## ğŸš€ Quick Start

### Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** >= 18.x ([Download](https://nodejs.org/))
- **npm** >= 9.x (comes with Node.js)

### Installation Steps

1. **Clone the Repository**

   ```bash
   git clone https://github.com/brenocrepaldi/mennu-client-frontend.git
   cd mennu-client-frontend
   ```

2. **Install Dependencies**

   ```bash
   npm install
   ```

   This will install all required packages defined in `package.json`.

3. **Start Development Server**

   ```bash
   npm run dev
   ```

   The application will start at [http://localhost:5173](http://localhost:5173)

4. **Open in Browser**
   - Navigate to `http://localhost:5173`
   - Start exploring the digital menu!

### Troubleshooting

**Build Errors?**

- Clear node_modules: `rm -rf node_modules && npm install`
- Check Node.js version: `node --version`
- Update npm: `npm install -g npm@latest`

**Port Already in Use?**

- Change port in `vite.config.ts` or kill the process using port 5173

## ğŸ—ï¸ Architecture Patterns

### Model-View-Page (MVP) Architecture

Mennu follows a clean architectural pattern where each feature module is organized into three distinct layers:

#### Model (`*.model.ts`) - Business Logic Layer

**Purpose:** Manages state, business logic, and data operations

**Responsibilities:**

- âœ… State management using React hooks and Zustand stores
- âœ… Data fetching and transformation
- âœ… Business rules and validations
- âœ… Event handlers and user interactions
- âœ… Computed values and derived state

#### View (`*.view.tsx`) - Presentation Layer

**Purpose:** Pure UI components that receive props from the model

**Responsibilities:**

- âœ… Rendering UI elements
- âœ… Displaying data passed as props
- âœ… No business logic or state management
- âœ… Composing smaller components
- âœ… Styling and animations

#### Page (`page.tsx`) - Integration Layer

**Purpose:** Connects Model and View, acts as the entry point for the route

**Responsibilities:**

- âœ… Imports and executes the model hook
- âœ… Passes model data to the view component
- âœ… Acts as the route component in React Router
- âœ… Minimal code, mainly composition

### Benefits of This Architecture

- **ğŸ”„ Separation of Concerns** - Clear boundaries between logic, presentation, and integration
- **ğŸ§ª Testability** - Easy to unit test models and views independently
- **â™»ï¸ Reusability** - Views can be reused with different models
- **ğŸ“– Maintainability** - Easy to locate and modify specific functionality
- **ğŸ‘¥ Team Collaboration** - Developers can work on different layers simultaneously
